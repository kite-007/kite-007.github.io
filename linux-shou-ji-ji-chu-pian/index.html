1. Linux目录结构

在Linux的世界里，一切都是文件

/（根）目录\*：根。
/boot（引导）目录\*：Linux启动的核心文件。
/swap（缓存）目录\*：与内存大小相同（交换分区）。
/bin目录：为Binary的缩写，存放着经常使用的命令和指令。
/sbin目录：s意为Super User的意思，这里存放着系统管理员所使用的系统管理程序。
/home目录\*：普通用户主目录。
/root目录\*：系统管理员主目录。
/lib目录：系统程序运行时需要用到这里的动态链接库，几乎所有的程序都要使用~win下的DLL文件。
/lost+found（隐藏）目录：一般为空，当非法关机后，这里会存在文件。
/etc目录\*：存放系统管理所需要的配置文件和目录，像安装了MySQL之后他的配置文件就在这里。
/usr目录\*：很重要的一个目录，大部分应用程序和文件都在这里~win的program files目录。
/proc（系统）目录：一个虚拟目录，存放系统内存映射，可以访问来获取系统信息。
/src（系统）目录：为service的缩写，存放服务启动之后需要提取的数据。
/sys（系统）目录：该目录安装了Linux2.6内核中新出现的一个文件系统 sysfs=》。
/tmp目录：存放临时文件。
/dev目录：把硬件设备用文件的形式存储，~win的设备管理器。
/media目录\*：Linux识别的一些设备，如U盘，光驱会被挂载到这里。
/mnt目录：外部临时挂载目录。
/opt目录：给主机额外安装软件所存放的目录（把要安装的软件放在这里进行解压安装）。
/usr/local目录\*：给主机额外安装软件所安装的目录，一般是通过编译源码方式安装的程序。
/var目录\*：存放着不断扩充更新或者经常修改的东西，像各种日志文件。
/selinux目录：全拼security-enhanced linux，SELinux是一种安全子系统，当启用服务时，该文件可以被看到，他控制程序只能访问特定的文件，有三种工作模式，可以自行设置。

2. Linux远程登陆与远程文件传输

一般工作时使用远程登陆与远程文件传输，需下载（Xshell与Xftp）.链接：https://www.netsarang.com/en/free-for-home-school/

  1. Xshell远程登陆
    • 使用Xshell来远程登陆到Linux，远程登陆使用ssh协议（22端口）。
    • ifconfig来查看网络信息，获取IP，并在主机终端进行ping，查看与主机的网络是否ping通。
    • 在Xshell中新建，输入要链接的Linux网络IP，选择一次性，输入账户密码，连接成功。
  2. Xftp远程文件传输
    • 与Xshell步骤一致
    • 左为Windows操作系统，右为Linux操作系统，可以鼠标右键进行相互传输
    • 处理Xftp中文乱码：菜单栏属性选项卡，选择选项，修改编码。

3. vi与vim的介绍与使用

Linux系统会内置vi文本编辑器，而vim则可以视为vi的加强版本，具有程序编程（代码补全，编译和错误跳转等）的能力，可以用字体颜色辨别代码语法的正确性。

  1. vi与vim介绍
    • vi与vim有三种模式：正常模式，插入模式和命令行模式
      • 正常模式（默认）:wq保存退出。
      • 插入模式，按下i, o, a, r等字母进入，按下Esc建会退出该模式进入到正常模式。
      • 命令行模式（低端），正常模式按下Shift与‘：’号进入，:w表示写入，:q表示保存，这之后加‘！’号表示强制的意思，与权限有关。
  2. vi与vim快捷键（常用）
    • yy：拷贝光标当前行，nyy则是拷贝当前光标所在的n行。
    • dd：删除光标当前行，ndd则是删除当前光标所在的n行。
    • 查找关键字：在命令行的模式下输入（/关键字）回车进行查找，按键n表示下一个。
    • 设置文件行号：在命令行模式下输入set nu，取消文件行号则是set nonu。
    • 移动光标：G为移动到最后一行，gg为移动到第一行，ngg则为移动到第n行。
    • 撤销一次动作（从正常模式来到插入模式后返回到正常模式）：u按键

4. 关机重启命令

  • 基本介绍：
    • shutdown -h now ; 立刻进行关机
    • shutdown -h 1 ; 1分钟之后关机，会给客户机发送提示信息
    • shutdown -r now ; 立刻进行重启
    • halt ; 关机
    • reboot ; 重启
    • sync ; 把内存的数据同步到磁盘
  • 细节：
    • 无论是重启或是关闭系统，首先都要运行sync命令，将内存中的数据写入到磁盘。
    • 目前在运行shutdown/halt/reboot等命令均以在关机前进行了sync。

5. 用户

  1. 切换用户：
    • 登录之后使用：su - 用户名，来更换用户，在多级用户下使用logout进行注销当前用户，1级无效。
  2. 添加用户：
    • root权限下使用useradd 用户名，来添加用户，默认该用户目录在/home目录下。
    • 也可以使用useradd -d 指定目录 用户名，来给创建的用户指定目录。
    • pwd，显示当前用户所在的目录
  3. 设置密码：
    • passwd 用户名，来给用户设置密码，如果不加用户，则是给当前用户设置。
  4. 删除用户：
    • userdel 用户名，保存该用户的家目录（该用户名的目录不是home）。
    • userdel -r 用户名，不保存该用户的家目录。
  5. 查询用户：
    • id 用户名，查询该用户所在的组。
    • whoami/who am i，查询当前所登录的用户。

6. 用户组

划分组可以使系统对多个用户进行统一的管理

  1. 新增组：
    • groupadd 组名，
  2. 删除组：
    • groupdel 组名，
  3. 增加用户时直接加上组：
    • useradd -g 用户组 用户名，如果创建用户不加组，系统会默认生成一个与用户名相同的组，并将该用户放在此组里面。
  4. 修改用户的组：
    • usermod -g 用户组 用户名，
    • usermod -d 目录名 用户名 ： 改变用户登陆的初始目录，用户需要有进入该目录的权限
  5. 用户和组相关的文件：
    • /etc/passwd：用户的配置文件，记录用户的各种信息。
    • 每行的含义：（用户名：口令：用户标识号（uid）：组标识号（gid）：注释性描述：主目录：登录Shell）
    • /etc/shadow：口令的配置文件
    • 每行的含义：（登录名：加密口令：最后一次修改的时间：最小时间间隔：最大时间间隔：警告时间：不活动时间：失效时间：标志）
    • /etc/group：组的配置文件，记录Linux下用户所包含的组信息
    • 每行的含义：（组名：口令：组标识号：组内用户列表）

7. 指定运行级别

CentOS7在/etc/inittab文件中有修改默认级别说明

运行级别说明

  • 0：关机
  • 1：单用户【找回丢失密码】
  • 2：多用户状态没有网络服务
  • 3：多用户状态有网络服务
  • 4：系统未使用保留给用户
  • 5：图形界面
  • 6：系统重启
  • 常用的运行级别为3和5，也可以指定默认运行级别
  • init X , X:0.1.2.3.4.5.6,切换不同的运行级别

8. 帮助指令

  1. man：获得帮助信息
    • man [命令或配置文件]：例如获取ls命令的帮助信息
  2. help：
    • help [命令]：获得Shell内置命令的帮助信息

9. 文件目录类

  1. pwd指令：显示当前工作目录的绝对路径。
  2. ls指令：ls [-选项]，列出目录。
    • -a：显示所有的文件和目录，包括隐藏的。
    • -l：以列表的形式显示信息（用户权限等）。
    • -h：文件大小显示，带单位。(要搭配-l使用)
    • -R：递归查询子目录
  3. cd指令：cd [参数]，切换到指定目录。
    • (cd ~)或者(cd 😃：回到自己的家目录。
    • (cd ..)：回到当前所在目录的上一级目录。
  4. mkdir指令：mkdir [-选项] [要创建的目录]。
    • -p：创建多级目录。
  5. rmdir指令：rmdir [-选项] [要删除的空目录]。
    • 这个指令删除的是空目录，如果目录下有内容则无法删除。
    • 如果要删除非空的目录，则需要使用 rm -rf [要删除的目录]，小心使用。
  6. touch指令：touch [文件名称]，创建一个空文件。
  7. 拷贝文件指令：cp [-选项] [哪个文件] [到哪里]。
    • -r：递归复制整个文件夹，如果存在相同的文件，不想出现提示信息，可以在cp前加\，\cp -r source dest。
  8. 移除文件或目录指令：rm [-选项] [要删除的文件或目录]。
    • -r：递归删除整个文件夹。
    • -f：强制删除不提示。
  9. 移动文件或目录，给文件重新命名：mv
    • mv [1] [2]，如果1与2在同一个目录，则1的名字会被重命名为2，反之，如果不在同一个目录，则1会被移动到2目录，可以移动整个目录。
  10. 查看文件内容：cat [-选项] [文件]
    • cat 与 vim的区别，cat只能浏览文件，不能修改文件，比较安全。
    • -n：显示行号
    • 为了浏览方便，一般会带上（管道命令 | more）（cat [-选项] [文件] | more）
    • 管道命令：前面的结果交给下一个指令处理
  11. more指令：more [要查看的文件]
    • 这是一个基于VI编辑器的文本过滤器，以全屏的方式按页显示文本文件的内容。
    • more设置了若干个快捷键：
      • 空白键：向下翻一页
      • Enter：向下翻一行
      • q：离开more模式，不再显示文件的内容
      • Ctrl+F：向下滚动一屏
      • Ctrl+B：返回上一屏
      • =：输出当前的行号
      • （:f）：输出文件名和当前的行号
  12. less指令：less [要查看的文件]
    • 与more的功能类似，但是更加强大。
    • 分屏查看文件的内容，他并不是将整个文件加载后显示，而是根据显示需要加载的内容，对于大型文件具有较高的效率
    • less快捷键：
      • 空白键：向下翻动一页
      • pagedown：向下翻动一页
      • pageup：向上翻动一页
      • /字符串：向下搜寻字符串的功能；n:向下查看，N:向上查看
      • ?字符串：向上搜寻字符串的功能；n:向上查看，N:向下查看
      • q：离开less模式，不再显示文件的内容
  13. echo指令：echo [-选项] [输出内容] > [目录]
    • 输出内容到控制台，输出内容重定向到该目录
  14. head指令：显示文件的开头的内容，默认情况下显示前10行的内容。
    • head [文件] ：查看文件的前10行的内容
    • head -n 5 [文件] ：查看文件的前5行的内容（5可以为任意数）
  15. tail指令：显示文件的尾部的内容，默认情况下显示后10行的内容。
    • tail [文件] ：查看文件的后10行的内容
    • tail -n 5 [文件] ：查看文件的后5行的内容（5可以为任意数）
    • tail -f [文件] ：实时监控文件更新的内容

  16. 指令与>>指令：>输出重定向和>>追加

    • ls -l >文件：列表的内容写入到文件中（覆盖）
    • ls -al >>文件：列表的内容追加到文件中（末尾）
    • cat 文件1 > 文件2：将文件1的内容覆盖到文件2
    • echo “内容”> 文件：（覆盖）
    • echo “内容”>> 文件：（追加）
  17. ln指令：软链接也成为符号链接，类似与win里的快捷方式，主要存放了链接其他文件的路径
    • ln -s [原文件或目录] [软链接名]：给原文件创建一个软链接
    • pwd指令查看文件目录时，显示的是软连接所在的目录
  18. history：显示历史命令
    • history n ：显示最近使用过的n个指令
    • !5：重新执行最近使用过的第5个指令

10. 时间日期类

11. 搜索查找类

  1. find指令
    • 从指定目录向下递归遍历各个子目录，将满足条件的显示在终端。
    • find [搜索范围] [选项]
    • -name <查询方式> 按照指定的文件名查找模式查找文件。
    • -user <用户名> 查找属于指定用户名所有文件。
    • -size <文件大小> 按照指定的文件大小查找文件（+n大于，-n小于，n等于，单位：K,M,G）。
  2. locate指令
    • 可以快速定位文件的路径，通过建立数据库的方式实现快速查找文件路径的目的。
    • 首次执行时，需要执行updatedb指令来创建locate数据库。
    • locate [文件]
  3. which指令
    • 可以查看某个指令的位置
    • which [ls]
  4. grep指令和管道符号 |
    • grep 表示过滤查找
    • “|”，表示将前一个命令的处理结果输出传递给后面的命令处理。
    • grep [选项] [查找内容] [源文件]
    • -n 显示匹配行及符号
    • -i 忽略字母大小写
    • 写法：
      • eg:cat /home/hello.txt | grep -n "yes",查找hello.txt文件中的yes,显示行号。
      • eg:grep -n "yes" /home/hello.txt,与上述效果相同。

12. 压缩和解压类

  1. gzip/gunzip指令：gzip为压缩命令，gunzip为解压命令
    • gzip [文件]，只能将文件压缩为.gz文件。
    • gunzip [文件].gz，解压缩文件命令。
  2. zip/unzip指令：zip用于压缩文件，unzip用于解压文件
    • zip [-选项] [压缩为XXX.zip] [要压缩的文件，包括本文件]
    • -r：递归压缩，压缩目录
    • unzip [-选项] [解压到此目录] [目录/XXX.zip]
    • -d<目录>：指定解压文件解压后形成的文件的存放目录。
  3. tar指令：将文件打包为.tar.gz
    • tar [-选项] [XXX.tar.gz] [要打包的内容，可用空格隔开]
    • tar [-选项] [要解压的文件.tar.gz] -C [要解压到的目录]
    • -c：产生.tar打包文件
    • -v：显示详情信息
    • -f：指定压缩后的文件名
    • -z：打包同时压缩
    • -x：解包.tar文件

13. 组管理和权限管理

组的基本介绍：Linux中，每个用户都必须存在一个组内，不能独立与组外。在Linux中每个文件有所有者，所在组，其他组的概念
1. 所有者
2. 所在组
3. 其他组
* =除文件的所有者和所在组的用户外，系统的其他用户都是文件的其他组=
4. 改变用户所在的组

  1. 文件/目录 所有者
    文件的所有者一般是该文件的创造者
    • 修改文件的所有者：
      • 查看文件的所有者：ls -ahl
      • 修改所有者：
        • chown [-选项] [用户名] [文件名]
      • 修改所有者和所在组
        • chown [-选项] [用户名]:[所在组] [文件名]
      • 修改文件所在的组：
        • chgrp [-选项] [组名] [文件名]
      • -R：递归更改文件属组，就是在更改某个目录文件的属组时，如果加上-R的参数，那么该目录下的所有文件的属组都会更改。

权限的基本介绍：
使用ll命令查看：
-rwxrwxrwx. 1 root root 314 3月 7 11:04 1.txt
权限说明 | 文件：硬链接数 或 目录：子目录数 | 用户 | 组 | 文件大小(字节) | 最后修改日期 | 文件名
看上述命令结果，-rwxrwxrwx. 这里一共10个位置
rwx可以用数字表示，r代表4，w代表2，x代表1，rwx=4+2+1=7

    * 0~9位说明：
       * 0位：确定文件的类型。
            * （l:软连接,相当于win里的快捷方式，d:表示目录,相当于win里的文件夹，c:字符设备文件,像鼠标键盘等，b:块设备,像硬盘，-:一般文件）。
        * 1~3：确定所有者拥有该文件的权限。rwx
        * 4~6：确定所属组拥有该文件的权限。rwx
        * 7~9：确定其他组拥有该文件的权限。rwx
            * （r:表示可读，w:表示可写，x:表示可执行）。
                * rwx作用到文件
                    * r:可以读取查看
                    * w:可以修改，不代表可以删除该文件，删除一个文件的前提条件是对该文件所在的目录有写权限 ，才能删除该文件。
                    * x:代表可以被执行
                * rwx作用到目录
                    * r:可以读取，ls查看内容
                    * w:可以修改，对目录内创建+删除+重命名目录
                    * x:可以进入该目录
  1. 修改文件权限 chmod命令,chmod [权限] [文件]
    1. +、-、=号修改权限
      • u:所有者，g:所有组，o:其他人，a:所有人(u、g、o的综合)
      • chmod u=rwx,g=rx,o=x [文件/目录名]
      • chmod o+w [文件/目录名]
      • chmod a-x [文件/目录名]
    2. 通过数字变更权限
      • chmod 777 [文件/目录名]

14. crond任务调度

任务调度：是指系统在某个时间执行的特定的命令或程序
任务调度分类：
1. 系统工作：有些重要的工作必须周而复始地执行。如病毒扫描等。
2. 个别用户工作：个别用户可能希望自行某些程序，比如对MySQL的数据备份。

  • 基本语法：crontab [-选项]
    • -e:编辑crontab定时任务
    • -l:查询crontab任务
    • -r:删除当前用户所有的crontab任务
    • service cront restart:重启任务调度
      • crond任务调度入门：
        • 设置任务调度文件：/etc/crontab
        • 设置个人任务调度。执行crontab -e命令
        • 接着输入任务到调度文件
        • 如：*/1 * * * * ls -l /etc/ > /tmp/to.txt
        • 意思是每小时的每分钟执行 ls -l /etc/ > /tmp/to.txt命令
          • */1 * * * *中五个占位符说明：
            1. 一小时中的第几分钟（0~59）
            2. 一天当中的第几小时（0~23）
            3. 一个月当中的第几天（1~31）
            4. 一年当中的第几个月（1~12）
            5. 一周当中的星期几（0~7）
          • 特殊符号的说明
            1. * ：代表任何时间。比如上述的第一个“*”，就代表一小时中每分钟都执行一次的意思。
            2. ,：代表不连续的时间。比如“0 8,12,16 * * *”，就代表在每天的8点0分，12点0分，16点0分都执行一次命令。
            3. -：代表持续的时间范围。比如“0 5 * * 1-6”，代表周一到周六的凌晨5点0分执行命令。
            4. */n：代表每隔多久执行一次。比如“*/10 * * * *”，代表每隔10分钟就执行一次命令

15. at定时任务

基本介绍：
1. at命令是一次性定时计划任务，at的守护进程atd会以后台模式运行，检查作业队列来运行。
2. 默认情况下，atd守护进程每60秒检查作业队列，有作业时，会检查作业的运行时间，如果时间与当前时间匹配，则运行次作业。
3. at命令是一次性定时计划任务，执行完一个任务后不再执行此任务了。
4. 在使用at命令时，一定要保证atd进程的启动，可以使用相关指令来查看。
5. ps -ef,查看运行中的进程。
6. atq,可以查看系统中没有执行的工作任务
7. atrm [编号],删除队列中未执行的任务

  • 基本语法：at [选项] [时间]，回车后输入要执行的命令，之后按Ctrl+D 结束at命令的输入
  • at命令选项:
    • -m ：当指定的任务被完成后，将给用户发送邮件，即使没有标准输出
    • -I(大写i) ：atq的别名
    • -d ：atrm的别名
    • -v： 显示任务将被执行的时间
    • -c： 打印任务的内容到标准输出
    • -V：显示版本信息
    • -q <队列>：使用指定的队列
    • -f<文件>：从指定文件读入任务而不是从标准输入读入
    • -t<时间参数>：以时间参数的形式提交要运行的任务
  • at指定时间的方法：
    1. 接受在当天的hh:mm(小时：分钟)式的时间指定。假如该时间已过去，那么就放在第二天执行。例如：04:00（如果已经过了当天的四点，那麽此进程就会放在第二天执行）。
    2. 使用midnight(深夜)，noon(中午)，teatime(饮茶时间，一般是下午4点)等比较模糊的词语来指定时间。
    3. 采用12小时计时制，即在时间后面加上AM(上午)或PM(下午)来说明是上午还是下午。例如：12pm。
    4. 指定命令执行的具体日期，指定格式为month day(月日)或mm/dd/yy(月/日/年)或dd.mm.yy(日.月.年)，指定的日期必须跟在指定时间的后面。例如：04:00 2021-03-1。
    5. 使用相对计时法。指定格式为：now + count time-units，now就是当前时间，time-units是时间单位，这里能够是minutes(分钟)、hours(小时)、days(天)、weeks(星期)。count是时间的数量，几天，几小时。例如：now + 5 minutes.
    6. 直接使用today(今天)、tomorrow(明天)来指定完成命令的时间。

19. 初识进程

  1. 介绍

    1. 在Linux中每个执行的程序都称为一个进程。每一个进程都分配了一个ID号（pid）进程号。
    2. 每个进程都以两种方式存在。前台与后台，所谓的前台进程就是用户目前屏幕上可以进行操作的。后台进程则是实际在操作，但由于屏幕上无法看到进程，通常使用后台方式运行。
    3. 一般系统的服务器都是以后台进程的方式存在，而且都会常驻在系统中。直到关机才结束。
    4. 程序与进程的关系：

  2. 显示系统执行的进程

    1. ps命令：ps命令是用来查看目前系统中，有哪些正在执行，以及他们执行的状况。可以不加任何参数。显示的信息选项：
  • ps -a:显示当前终端所有的用户信息
  • ps -u:以用户的格式显示进程信息
  • ps -x:显示后台进程运行的参数
  • 字段：USER，说明：进程执行用户
  • 字段：PID，说明：进程识别号
  • 字段：%CPU，说明：占用cpu百分比
  • 字段：%MEM，说明：占用实际物理内存百分比
  • 字段：VSZ，说明：占用虚拟内存实际情况
  • 字段：RSS，说明：占用物理内存实际情况
  • 字段：TTY，说明：终端机号信息
  • 字段：STAT，说明：进程运行状态,s表示sleep,r表示run,d表示短期等待,z表示僵死进程（未释放内存）等等
  • 字段：START，说明：进程开始时间
  • 字段：TIME，说明：此进程所消耗CPU的时间
  • 字段：COMMAND（CMD），说明：启用进程的指令或者参数，过长会被截断
  • 字段：CMD，说明：正在执行的命令或进程名
  • ps -e:显示所有进程
  • ps -f:全格式
  • 字段：PPID，说明：父进程识别号
  • 字段：C，说明：cpu用于计算执行优先级的因子。数值越大，表明进程是cpu密集型运算，执行优先级会减低；数值越小，表明进程是I/O密集型运算，执行优先级会提高
  • 字段：STIME，说明：进程的启动时间

  1. 终止进程kill和killall

    1. 基本语法：

      2. killall 进程名称 （通过进程名称杀死/终止进程，也支持通配符，这在系统因负载过大而变得很慢的时候有用）
      3. -9：表示强迫进程立即停止

        3. 重启服务：/bin/systemctl start sshd.service
      3. 终止多个gedit ：killall
        1. 打开多个文本编辑器，模拟
        2. killall gedit，进程终止包括子进程
      4. 强制终止一个终端

20. 服务管理（service）与端口打开关闭（防火墙）

服务的本质就是进程，但是是运行在后台的，通常都会监听某个端口，等待其他程序的请求，像（mysqld，sshd，防火墙等等），因此又称为守护进程。

  1. service管理指令

    1. service [服务名] [start | stop | restart | reload| status]
    2. CentOS 7.0 之后 一些服务不再用service，而替换为systemctl
    3. 还在使用service命令的服务，可以在 /etc/init.d中查看，ls -l /etc/init.d 其中的绿色字体。
    4. 查看服务：（1）使用(setup)查看全部系统服务，(*)表示自启动，光标放入按空格键取消自启动（2）上述ls -l /etc/init.d可以查看部分服务。

  2. 服务器的运行级别（runlevel）前面已经简单提到过有七种运行级别

    • 0：关机，系统停机，系统默认运行级别不能设为0，否则不能正常启动
    • 1：单用户【找回丢失密码】工作状态，有root权限，禁止远程登陆
    • 2：多用户状态没有网络服务，因为没有FNS
    • 3：多用户状态有网络服务，有FNS，登陆后进入控制台命令模式
    • 4：系统未使用保留给用户
    • 5：(图形界面)，X11控制台，登陆后进入图形GUI模式
    • 6：(系统重启)系统正常关闭并重启，默认运行级别不能设为6，否则不能正常开机。
    • 开机流程说明：开机——》BIOS——》/boot——》systemd进程1——》运行级别——》运行级对应的服务

  3. 服务级别命令：

    1. 查看默认级别，systemctl get-default
    2. 设置默认级别为级别3，systemctl set-default [multi-user.target]，/etc/inittab中有说明
    3. chkconfig命令：
      1. 介绍：
        1. 通过此命令可以给服务的各个运行级别设置 自启动/关闭
        2. chkconfig 指令管理的服务在/etc/init.d中查看，与system相同，在centos7之后很多服务用systemctl管理
      2. 基本语法：(chkconfig重新设置服务后自启动或关闭，需要reboot生效)
        1. 查看服务：chkconfig --list [| grep XXX]
        2. chkconfig 服务名 --list
        3. chkconfig --level 5 [服务名] on/off
    4. systemctl命令：
      1. systemctl管理命令：
        1. systemctl [start | stop | restart | status] [服务名]
        2. systemctl管理的服务在/user/lib/systemd/system中查看（名称）
      2. 设置服务的自启动状态：
        1. systemctl list-unit-files [| grep 服务名]，查看服务开机启动状态，grep可以进行过滤，服务名称可以不带.service。
        2. systemctl enable 服务名，设置服务开机启动，centos7之后,级别被简化为3和5，表示服务在3和5级别开机自启动。
        3. systemctl disable 服务名，关闭服务开机启动，centos7之后,级别被简化为3和5，表示服务在3和5级别开机关闭。
        4. systemctl is-enabled 服务名，查询某个服务是否是自启动的
        5. 细节讨论：
          1. 关闭或者启用防火墙之后，立即生效。【telnet [IP] [端口]访问测试 】
          2. 这种方式只是临时生效。当重启系统之后，还是回归以前对服务的设置。
          3. 如果希望设置某个服务自启动或关闭永久生效，要使用systemctl [enable | disable] 服务名
    5. firewall指令：可以打开或者关闭指定的端口。
      1. 打开端口：firewall-cmd --permanent --add-port=[端口号/协议]都要有
      2. 关闭端口：firewall-cmd --permanent --remove-port=[端口号/协议]都要有
      3. 重新载入才能生效：firewall-cmd --reload
      4. 查询端口是否开放：firewall-cmd --query-port=[端口号/协议]都要有

21. 动态监控进程（交互显示执行的进程）

top命令与ps命令相似。它们都用来显示正在执行的进程。top与ps最大的不同之处就在于，top命令在执行一段时间是可以更新正在运行的进程，并且可以进行交互操作。

  1. top命令
    1. top [选项]
      1. -d [秒数]，指定top命令每隔几秒更新，默认是3秒
      2. -i，使top不再显示任何闲置或者僵死进程
      3. -p，通过指定监控进程ID来仅仅监控某个进程的状态
    2. 交互操作说明
      1. P:以CPU使用率排序，默认就是此项
      2. M:以内存的使用率排序
      3. N:以PID排序
      4. q:退出top
      • 监视特定用户
        1. top：输入此命令，按Enter，查看执行的进程
        2. u：然后输入“u”回车，再输入用户名，即可
      • 终止指定的进程
        1. top：输入此命令，按Enter，查看执行的进程
        2. k：然后输入“k”回车，再输入要结束的进程ID号

      • 案例： rpm -qf /etc/passwd
      • 案例：rpm -qf /root/install.log
    6. rpm -e [RPM包的名称]：卸载rpm包
      • 如果有其它的软件包依赖于要卸载的软件包，则会产生错误信息，无法卸载。
      • 如：rpm -e XXX ，
      • 如果要强制删除XXX则可以在后面加参数 --nodeps，就可以强制删除，但是如果这样做，依赖于该软件包的程序可能无法运行。
      • 如：rpm -e --nodeps XXX
    7. rpm -ivh [RPM包全路径名称]：安装rpm包
      • i=install安装
      • v=verbose提示
      • h=hash进度条
      • RPM包：再光盘中有大量RPM包，把找到的包路径放到/opt/目录下
  • yum
    yum是一个shell前端软件包管理器。基于RPM包管理，能够从指定的服务器自动下载RPM包并且安装，可以自动处理依赖性关系，并且一次性安装所有依赖的软件包
  1. yum的基本指令：
    1. 查询yum服务器是否有需要安装的软件
      • yum list | grep xx软件列表
    2. 安装指定的yum包
      • yum install XXX 下载安装
    3. 应用实例：
      • 使用yum的方式安装firefox
