# Docker应用部署（CentOS-7）

2021-03-27

  • 1. MySQL部署
  • 2. Tomcat部署
  • 3. Nginx部署
(Docker)

  • 容器内的网络服务和外部机器不能直接通信
  • 外部机器可以和宿主机可以进行直接通信
  • 宿主机和容器可以进行直接通信
  • 操作（端口映射 -p）：假设容器内打开了端口3306，那麽在宿主机上面打开端口3307，将端口3306与3307进行一个映射，之后外部机器访问3307端口就相当于访问了容器内部的3306端口。

1. MySQL部署

  • 需求：在Docker容器中部署MySQL，并通过外部MySQL客户端操作MySQL Server
  1. 搜索MySQL镜像
docker search MySQL
  1. 拉取MySQL镜像
docker pull MySQL:5.6
  1. 创建启动容器
# 在root目录下创建mysql目录用于存储mysql数据信息
mkdir /root/mysql
cd /root/mysql

#创建并运行
docker run -id \
-p 3307:3306 \
--name=c_mysql \
-v $PWD/conf:/etc/mysql/conf.d \
-v $PWD/logs:/logs \
-v $PWD/data:/var/lib/mysql \
-e MYSQL_ROOT_RASSWORD=123456 \
mysql:5.6
  • -id：以守护进程的方式运行。
  • -p：端口映射，宿主机的3307端口与容器的3306端口进行映射，一般情况下宿主机端口与容器端口相同，即3306：3306。
  • -v $PWD/conf:/etc/mysql/conf.d ：将主机当前目录下的conf/my.cnf挂载到容器的/etc/mysql/my.cnf。配置目录
  • -v $PWD/logs:/logs ：将主机当前目录下的logs目录挂载到容器的/logs。日志目录
  • -v $PWD/data:/var/lib/mysql ：将主机当前目录下的data目录挂载到容器的/var/lib/mysql。数据目录
  • -e MYSQL_ROOT_RASSWORD=123456 ：初始化root用户密码

$PWD：我们先创建了MySQL目录，之后进入到MySQL目录，就相当于/root/mysql

  1. 操作MySQL
mysql -uroot -p123456

2. Tomcat部署

  • 需求：在Docker容器中部署Tomcat，并通过外部访问Tomcat
  1. 搜索Tomcat镜像
docker search Tomcat
  1. 拉取MySQL镜像
docker pull Tomcat:9.0
  1. 创建启动容器
# 在root目录下创建tomcat目录用于存储tomcat数据信息
mkdir /root/tomcat
cd /root/tomcat

#创建并运行
docker run -id \
-p 8080:8080 \
--name=c_tomcat \
-v $PWD:/usr/local/tomcat/webapps \
tomcat:9.0
  • -id：以守护进程的方式运行。
  • -p：端口映射，宿主机的8080端口与容器的8080端口进行映射，一般情况下宿主机端口与容器端口相同，即8080：8080。
  • -v $PWD:/usr/local/tomcat/webapps \：将主机的当前目录挂载到容器的webapps，项目文件存放在webapps中。

3. Nginx部署

  • 需求：在Docker容器中部署Nginx，并通过外部访问Nginx
  1. 搜索Nginx镜像
docker search nginx
  1. 拉取MySQL镜像
docker pull nginx
  1. 创建启动容器
# 在/root目录下创建nginx目录用于存储nginx数据信息
mkdir ~/nginx
cd ~/nginx
mkdir conf
cd conf
# 在~/nginx/conf/下创建nginx.conf文件,粘贴下面内容
vim nginx.conf
  • nginx.conf文件需要提前准备好
user  nginx;
worker_processes  1;

error_log  /var/log/nginx/error.log warn;
pid        /var/run/nginx.pid;


events {
    worker_connections  1024;
}


http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log  /var/log/nginx/access.log  main;

    sendfile        on;
    #tcp_nopush     on;

    keepalive_timeout  65;

    #gzip  on;

    include /etc/nginx/conf.d/*.conf;
}

#创建并运行
docker run -id \
-p 80:80 \
--name=c_nginx \
-v $PWD/conf/nginx.conf:/etc/nginx/nginx.conf \
-v $PWD/logs:/var/log/nginx \
-v $PWD/html:/usr/share/nginx/html \
nginx
